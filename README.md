# 2026 World Cup Prediction Simulation

- **Data source**: https://www.eloratings.net
- **Model**: Poisson regression & Monte Carlo Simulation
- **Input Data**:
  - `data/elo_ratings.json` (팀별 초기 Elo 레이팅)
  - `data/groups.json` (48개국 12개 조 편성표)
  - `data/actual_results.json` (실제 경기 스코어 기록)

## 프로젝트 개요 및 주요 기능
본 프로젝트는 Elo 레이팅과 포아송 분포를 기반으로 2026년 월드컵(48개국 체제) 전체 과정을 10,000번 반복 시뮬레이션하여 각 국가의 토너먼트 진출 확률을 예측합니다.

### 1. 단일 경기 결과 예측 (`src/poisson.py`)
- 양 팀의 전력(Elo 레이팅) 차이를 바탕으로 각 팀의 예상 득점(Lambda)을 산출합니다.
- `numpy.random.poisson`을 사용하여 예상 득점을 실제 축구 스코어 형태(예: 2-1, 0-0)로 무작위 추출합니다.

### 2. 조별 리그 진행 (`src/simulation.py`)
- 조별(A~L조) 4개 팀이 서로 1번씩 맞붙는 풀리그(조당 6경기)를 진행합니다.
- 승점(승 3, 무 1, 패 0), 골득실차, 다득점을 계산하여 그룹별 1~4위 순위를 정렬합니다.

### 3. 32강 대진표 및 토너먼트 진행
- 각 조 1, 2위 24팀과 3위 중 상위 8팀(와일드카드)을 선별해 총 32팀을 구성합니다.
- 성적순 시드 배정으로 대진표를 만들고 단판 승부를 시뮬레이션합니다. 90분 무승부 시 30분 연장전(Extra Time)을 추가 진행하며, 연장전 후에도 동점인 경우 Elo 승률 기반의 확률적 승부차기(PK)로 반드시 승자를 결정합니다.

### 4. 실시간 데이터 자동 수집 및 혼합 반영 (`fetch_data.py`)
- eloratings.net에서 월드컵 참가국의 최신 실시간 ELO 레이팅과 완료된 실제 경기 결과를 동시에 수집하여 로컬 데이터(`elo_ratings.json`, `actual_results.json`)를 자동으로 최신화한 직후, Wikipedia에서 최신 징계(출장정지) 정보를 자동으로 연쇄 동기화하여 [data/absences.json](file:///Users/gwonchangbin/projects/worldcup_prediction_2026/data/absences.json)을 갱신합니다.
- **조별 리그 실시간 반영 & 대진 동적 고정**: 실제 치러진 경기는 조별 리그 시뮬레이션 시 실제 점수로 고정(Override) 대입됩니다. 조별 리그가 완전히 종료된 조는 100% 실제 성적에 따라 순위가 결정되며, 피파 공식 32강 매칭 대진 규칙에 맞추어 실제 진출팀이 지정된 32강 트리 슬롯에 자동으로 고정 배치됩니다.
- **토너먼트 실제 결과 고정 (Override)**: 32강 이후 토너먼트 경기도 실제 결과가 등록되면 시뮬레이션을 무시하고 실제 최종 스코어와 실제 승리 국가(Winner)를 강제로 다음 라운드에 진출시키며 대진을 실시간으로 고정해 나갑니다.


### 5. 개별 국가 간 승부 예측기 (`predict_match.py`)
- 사용자가 선택한 두 국가의 현재 ELO 레이팅과 포아송 분포를 기반으로 개별 맞대결의 승/무/패 확률을 산출하고, 넘파이(NumPy) 벡터 연산 기반의 1,000,000회 시뮬레이션을 통해 가장 발생 확률이 높은 대표 예상 스코어를 정밀하게 도출합니다.

### 6. 몬테카를로 시뮬레이션 (`main.py`)
- 앞서 구현한 "월드컵 전체 과정"을 10,000번 반복하여 각 국가의 진출 확률(%)을 통계로 추출합니다.

### 7. Streamlit AI 실시간 웹 대시보드 (`app.py`)
- Python 기반 웹 프레임워크인 Streamlit을 활용하여 인터랙티브한 분석 대시보드를 시각화합니다.
- **실시간 부상 및 전력 누수 관리 (사이드바)**: 실시간으로 부상 선수를 스쿼드에 등록하거나 복귀 처리할 수 있으며, 이 결과는 대시보드 내 모든 예측 시뮬레이션에 즉시 동적 반영됩니다.
- **경기 일정 및 AI 승무패 예측 (Tab 1)**: 2026 월드컵 전체 104경기의 일정 필터링 및 ELO/피로도 격차를 결합한 AI 최빈 예상 스코어 및 3대 결과 확률(승/무/패) 게이지바 시각화.
- **조별 리그 실제 진행도 (Tab 2)**: 실제 치러진 경기 결과를 바탕으로 실시간 승점과 골득실을 자동 연산한 12개 조(A~L)의 순위표를 즉시 시연합니다 (불필요한 이모티콘을 모두 배제한 깔끔한 UI 제공).
- **토너먼트 대진표 시각화 (Tab 3)**: Left-Center-Right 대칭형 브래킷 대진표(32강~결승)를 한 화면에 트리로 렌더링하며, 각 경기를 클릭하면 상세 분석 모달 팝업이 노출됩니다.
- **1대1 가상 매치 시뮬레이터 (Tab 4)**: 두 팀의 매치 조건(홈 우위, 휴식일 차이, 비행 피로도)을 사용자가 직접 제어하여 실시간 ELO 기반 가상 경기 결과를 정밀하게 예측합니다.
- **몬테카를로 시뮬레이터 (Tab 5)**: 전체 대회를 최대 10,000회 시뮬레이션하여 얻은 단계별 생존 확률 통계를 내림차순 리스트로 시연하며, UI 개선을 통해 조절 슬라이더와 실행 버튼을 세련되게 정렬했습니다.

## 수치 산출 방식 및 공식 (Mathematics & Formulas)

본 시뮬레이션은 **Elo 레이팅**과 **포아송 분포(Poisson Distribution)** 두 가지 수학적 모델을 결합하여 경기 결과를 예측합니다.

### 1. 기대 승률 산출 및 레이팅 업데이트 (Elo Rating System)
두 팀의 Elo 점수 차이를 바탕으로 각 팀의 승리 기대값(확률)을 산출합니다. (`src/elo.py`)

- **공식**:
  $$E\_A = \frac{1}{1 + 10^{(R\_B - R\_A) / 400}}$$
  - $E\_A$: 팀 A의 기대 승률 (0 ~ 1)
  - $R\_A$: 팀 A의 Elo 점수, $R\_B$: 팀 B의 Elo 점수
- **설명**: 두 팀의 점수차가 400점일 때, 점수가 높은 팀의 승률은 약 90.9%로 계산됩니다.
- **월드컵 가중치 (K-factor)**: 실제 경기 결과(`fetch_data.py`) 반영 시, 월드컵 본선 경기의 공식 중요도인 **`K-factor = 60`**을 대입하여 경기 결과가 각 팀의 전력(Elo)에 즉각적이고 뚜렷하게 반영되도록 업데이트합니다.

### 2. 팀별 예상 득점(Lambda, $\lambda$) 산출
Elo 기반 기대 승률을 축구 경기의 팀 평균 득점(Base Goals)을 활용해 '예상 득점 수치($\lambda$)'로 변환합니다. (`src/poisson.py`)

- **공식**:
  $$\lambda\_A = \text{Base Goals} \times \left( \frac{E\_A}{1 - E\_A} \right)^{0.376}$$
  $$\lambda\_B = \text{Base Goals} \times \left( \frac{1 - E\_A}{E\_A} \right)^{0.376}$$
  - $\lambda\_A$, $\lambda\_B$: 팀 A와 팀 B의 예상 득점 (포아송 분포의 모수)
  - $\text{Base Goals}$: 한 팀의 평균 득점 기준값 (본 프로젝트에서는 **1.35골** 적용, 경기당 총합 2.7골)
  - $E\_A$: 팀 A의 기대 승률
- **설명**: 기존 선형 공식($\lambda\_A = 3.0 \times E\_A$)은 두 팀의 예상 득점 합이 3.0골로 고정되어, 강팀-약팀 경기에서 강팀의 다득점이 억제되고 무승부 확률이 지나치게 낮아지는 한계(20% 미만)가 있었습니다. ELO 기대 승률의 비율(Odd)에 거듭제곱($0.376$)을 취한 비선형 공식으로 개선하여, 무승부 확률이 실제 월드컵 데이터 수준(평균 약 25.8%)으로 수렴하고 강팀-약팀 매치에서 다득점이 유연하게 발생하도록 조율했습니다.

### 3. 실제 스코어 확률 산출 (Poisson Distribution)
산출된 예상 득점($\lambda$)을 포아송 확률 질량 함수에 대입하여, 한 팀이 특정 경기에서 $k$골을 넣을 확률을 계산합니다. 축구 경기의 득점은 독립적인 사건으로 가정합니다.

- **공식**:
  $$P(X = k) = \frac{\lambda^k e^{-\lambda}}{k!}$$
  - $P(X = k)$: 한 팀이 정확히 $k$골을 득점할 확률
  - $\lambda$: 팀의 예상 득점, $e$: 자연상수 (약 2.718)
  - $k$: 득점 수 (0, 1, 2, 3...)
- **설명**: 시뮬레이션 단계에서는 위 수식을 확률 분포로 갖는 `numpy.random.poisson(lambda)` 함수를 사용하여 실제 스코어(예: 2-1, 0-0)를 난수로 추출해 경기를 진행합니다.

## 선수 결장(부상/징계) 및 스쿼드 가치 기반 동적 보정 시스템 (Absent Players & HHI Adjustment)

본 시뮬레이션은 핵심 주전 선수의 부상 및 카드 징계(출장정지)로 인한 전력 누수를 단순 수동 등급 및 정적 뎁스 대신, **참가국 전체 스쿼드의 선수별 시장 가치(Market Value)** 및 **스타 의존도(HHI 지수)** 데이터를 기반으로 동적으로 계산합니다.

### 1. 스쿼드 및 결장 데이터 구축 (`fetch_injuries.py`, `fetch_suspensions.py`)
- **실시간 부상 데이터 (`fetch_injuries.py`)**: Wikipedia 공식 [2026 FIFA World Cup squads](https://en.wikipedia.org/wiki/2026_FIFA_World_Cup_squads) 페이지에서 48개국 스쿼드 정보를 수집하고, 엔트리 이탈 정보를 파싱하여 원조 부상 선수를 [data/absences.json](file:///Users/gwonchangbin/projects/worldcup_prediction_2026/data/absences.json)에 기록합니다.
- **실시간 징계 데이터 (`fetch_suspensions.py`)**: 대회 중 발생하는 퇴장(레드카드) 및 경고 누적 정보를 Wikipedia 메인 문서의 `Discipline` 섹션에서 실시간 크롤링하여 `data/absences.json`에 구조화된 형식으로 기록합니다.
- **실시간 징계 자동 복귀 (Auto-Restoration)**: 징계 선수가 등록되는 시점에 해당 국가의 누적 경기 수($N$)를 기준으로 복귀 기준이 될 경기 수(`served_at_count = N + 정지 경기 수`)를 산정합니다. 대시보드를 로딩할 때 실제 경기 결과(`data/actual_results.json`)가 추가되어 국가의 누적 경기 수가 복귀 기준 이상이 되면 자동으로 명단에서 제거하고 전력을 정상 복구시킵니다.

### 2. 포지션별 전력 비중 (Value Share) 계산
부상/징계 선수의 포지션(GK, DF -> 수비 / MF, FW -> 공격)에 해당하는 스쿼드 내 총 가치 대비 해당 선수의 가치 비율을 구합니다.
$$S\_p = \frac{\text{Value}\_p}{\sum_{i \in \text{Position}} \text{Value}\_i}$$
- *예시*: 한국 공격진 전체 가치가 1억 유로이고 손흥민 선수의 가치가 5천만 유로라면, 손흥민 결장 시 공격진 내 전력 비중($S\_p$)은 **50%**가 됩니다.

### 3. 팀의 스타 의존도 및 뎁스 감쇄 (Concentration Index)
스쿼드의 가치 분산도를 나타내는 **허핀달-허쉬만 지수(HHI)**를 사용하여 팀의 에이스 의존도 계수($D\_{\text{team}}$)를 계산합니다.
$$H\_{\text{team}} = \sum\_{i=1}^{26} \left( \frac{\text{Value}\_i}{\text{Total Value}\_{\text{team}}} \right)^2$$
- **의존도 계수**: $D\_{\text{team}} = 0.2 + 0.8 \times \text{Normalized } H\_{\text{team}}$ (최소 0.2 ~ 최대 1.0 범위로 정규화)
- **최종 전력 누수율 (Reduction)**:
  $$\text{Reduction}\_{\text{pos}} = \sum\_{p \in \text{Absent}} S\_p \times D\_{\text{team}}$$
- **효과**: 백업이 매우 탄탄해 HHI가 낮은 팀(예: 프랑스, 잉글랜드)은 에이스 결장 시 전력 누수가 최소화되는 반면, 스타 플레이어 한두 명에 가치가 몰려 HHI가 높은 팀(예: 대한민국, 이집트)은 결장 시 타격이 극대화됩니다.

### 4. 최종 보정율 적용 규칙
- 공격 배율 = $\max(0.5, 1.0 - \text{공격 누수율})$ (내 득점력을 낮춤)
- 수비 배율 = $\min(2.0, 1.0 + \text{수비 누수율})$ (상대 득점력을 높임)
- 최종 예상 득점 ($\lambda\_{A(\text{최종})}$) 계산 시 양 팀의 결장 보정 배율을 교차 곱해 반영합니다.

## 월드컵 대진 환경 변수 보정 시스템 (Home Advantage, Rotation, Rest & Travel Fatigue)

시뮬레이션의 사실성을 극대화하기 위해, 실제 월드컵 경기 일정 및 지리적 요건을 모델링하여 다음 4가지 동적 보정을 경기 단위로 적용합니다.

### 1. 공동 개최국 홈 우위 (Host Premium)
공동 개최국인 `USA`, `Mexico`, `Canada`가 경기를 치를 때, 홈 관중 및 적응 이점을 반영하여 임시 ELO 점수 **+40점**을 더해 계산합니다. (기존 +70점에서 역대 월드컵 개최국 ELO 통계치인 +30~+50점에 맞추어 하향 조정하였으며, 개최국끼리의 맞대결일 경우 효과는 서로 상쇄됩니다.)

### 2. 조별 리그 3차전 로테이션 페널티 (Squad Rotation)
조별 리그 순서를 공식 매치 라운드(1~2라운드 선 진행 $\rightarrow$ 3라운드 진행)대로 정렬하여 수행하며, 2라운드 종료 시점에 2승(승점 6점)으로 32강 토너먼트 진출을 확정한 팀은 3라운드 경기 시 **공격 람다 0.8배 페널티(득점력 20% 감소)**를 임시 적용합니다.

### 3. 토너먼트 휴식일 격차 보정 (Rest Days Gap)
32강~결승전까지 각 팀의 매치 완료 날짜를 기준으로 다음 경기까지의 휴식 일수(Rest Days)를 구하여, 상대보다 더 오래 휴식한 일수(하루당 **+5 ELO**)를 누적 가산하며 최대 **+30 ELO**로 제한(캡 적용)합니다.
* **공식**: `rest_bonus = min(abs(rest_days_diff) * 5, 30)`

### 4. 대륙 이동 피로도 보정 (Travel Fatigue)
북미 대륙의 16개 개최 도시를 지리적 위치와 시차에 따라 **5개 권역(Region 1~5)**으로 분류하여 개최 도시 간 이동에 따른 피로 누수를 반영합니다.
* **Region 1 (서부)**: 밴쿠버, 시애틀, 샌프란시스코, LA
* **Region 2 (멕시코/중서부)**: 과달라하라, 몬테레이, 멕시코시티
* **Region 3 (중부)**: 댈러스, 휴스턴, 캔자스시티
* **Region 4 (남동부)**: 마이애미, 애틀랜타
* **Region 5 (동부)**: 토론토, 보스턴, 필라델피아, 뉴욕/뉴저지

**피로도 감쇄 규칙**:
* 같은 권역 내 이동 (권역 차이 = 0): 페널티 없음 (0%)
* 인접 권역 이동 (권역 차이 = 1, 2): 공격 람다 **-1.5%** 감쇄
* 대륙 횡단 이동 (권역 차이 $\ge$ 3): 공격 람다 **-3.0%** 감쇄
* **최종 예상 득점 ($\lambda$)**:
  $$\lambda\_{A(\text{최종})} = \lambda\_A \times (\text{A팀 결장 공격보정}) \times (\text{B팀 결장 수비보정}) \times (1.0 - \text{A팀 이동 피로도})$$

## 실행 방법

```bash
# 1. 가상 환경 활성화 (선택)
source venv/bin/activate

# 2. 의존성 패키지 설치
pip install -r requirements.txt

# 3. 실시간 ELO 레이팅, 경기 결과 및 징계(출장정지) 정보 동기화
PYTHONPATH=. python3 fetch_data.py

# 4. Streamlit 웹 대시보드 앱 구동 (추천)
venv/bin/streamlit run app.py

# 5. 몬테카를로 시뮬레이션 실행 (약 10~30초 소요)
PYTHONPATH=. python3 main.py

# 6. 개별 두 국가 간 승부 예측 (선택)
# 인자를 생략하고 실행하면 터미널에서 대화식으로 팀명을 물어봅니다.
PYTHONPATH=. python3 predict_match.py "South Korea" "Mexico"
```
